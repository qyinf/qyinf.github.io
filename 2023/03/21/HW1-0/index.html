一、题目概述

本次作业的主要要求，就是对一个输入的表达式进行化简
以下是我对表达式结构的个人理解，就不粘贴原题了
表达式(expression)的构造如下：

expression =  term ± term ± term ± term…

即由若干项(term)相加或者相减组成，第一个项的符号可以带也可以不带
而term的构造如下:

term = (±) factor * factor *factor * factor…

即由若干因子(factor)相乘组成，且题目中要求每个term的前面可以带一个正号或负号，也可以不带
而因子的类别随着作业的迭代逐渐变多
第一次作业的因子，只有
1.常数因子：一个可以带符号，可以带前导零的数字，合法的形式有+000008,-5,1111111111111111111111111111111111111111111
2.幂函数因子：字母**常数因子或字母，常数因子只能是正数且不能太大，合法的形式有x**+0003,y**5，z
3.表达式因子：(表达式)** 常数因子 或 (表达式) ，常数因子只能是正数且不能太大，合法的形式有 (x),(x)**3,(-x+3)**+005
第二次作业的因子，增加了两个：
4.三角函数因子：sin/cos(表达式)**常数因子，或sin/cos(表达式),常数因子只能是正数且不能太大，合法的形式有sin(x)**+5,cos(-y)
5.自定义函数因子：
在开头的时候，先定义函数：f(x,y,z)=表达式，比如f(x)=x**2,f(z,x)=x+z
函数的名字只有f，g，h，且不能重复出现，因此可以知道最多有三个自定义函数
调用函数的格式是,f(因子，因子，因子)
第三次作业的时候，增加了求导因子
dx | dy | dz (表达式) ，合法的格式有dx (x),dy(y+z+x)等

我们可以注意到，这些因子中包含了很多嵌套，每一次作业的嵌套规定如下：
第一次作业
表达式因子里面的表达式，不可以包含表达式因子，也就是说，不允许出现括号的嵌套
第二次作业
1.允许出现任何形式的括号嵌套
2.函数的定义不可以引用其他函数，比如不可以出现f(x)=g(x)这种情况，函数的定义也不可以引用自己
第三次作业
函数的定义可以引用其他出现过的函数，比如先定义了g，又定义了f，这个时候f的定义可以引用g

这个函数的思路很简单，就是根据+-号个数的奇偶性进行化简的：

public static String processStr(String ref) {
        String ans = "";
        ref = ref.replaceAll(" ","");
        int i = 0;
        while (i < ref.length()) {
            if (ref.charAt(i) == '+' || ref.charAt(i) == '-') {
                int plus = 0;int minus = 0;int j;
                if (ref.charAt(i) == '+') {
                    plus = 1; }
                else if (ref.charAt(i) == '-') {
                    minus = 1; }
                for (j = i + 1;j < ref.length();j++) {
                    if (ref.charAt(j) == '+') {
                        plus++; }
                    else if (ref.charAt(j) == '-') {
                        minus++; }
                    else {
                        break; } }
                if (minus % 2 == 0) {
                    ans = ans + "+"; }
                else {
                    ans = ans + "-"; }
                i = j; }
            else {
                ans = ans + ref.charAt(i);
                i++; } }
        return ans;
    }

f(y,x)=x-2*y //函数定义
f(x,y)//函数调用

求导因子

增加了一个求导因子，求导因子应该作为一个factor，而非expression或Item

求导方法的书写其实特别简单：

对一个poly求导就是对它的每个mono求导再加起来

对一个mono求导得到的结果应该是一个poly

对每一个mono求导的思路如下,以x为例，展示思路

public Poly monoDevX() {
        //每一个单项式求导以后会变成很多多项式
        BigInteger ref = new BigInteger(Integer.toString(this.xx));
       //首先不管sin和cos，对字母部分求导
        Mono mono1 = new Mono();
        mono1.set(this.xx - 1,this.yy,this.zz,this.sign,this.flg,ref.multiply(this.num));
        mono1.setSinContent(this.sinContent);
        mono1.setCosContent(this.cosContent);
        Poly ans = new Poly();
        ans.addMono(mono1);
        //然后对sin求导，符号不会发生改变，sin少了1，cos多了1
        for (int i = 0;i < this.sinContent.size();i++) {
            Mono monoI = new Mono();
            monoI.addCosContent(this.sinContent.get(i));
            for (int j = 0;j < this.sinContent.size();j++) {
                if (j != i) {
                    monoI.addSinContent(this.sinContent.get(j)); }
            }
            for (int j = 0;j < this.cosContent.size();j++) {
                monoI.addCosContent(this.cosContent.get(j));
            }
            //这里是求导法则的调用，对sin里面求导，再与外面的相乘
            monoI.set(this.xx,this.yy,this.zz,this.sign,this.flg,this.num);
            Poly tmp = new Poly();
            tmp.addMono(monoI);
            Poly tmp1 = new Poly();
            tmp1.addPoly(this.sinContent.get(i),1);
            tmp1.devX();
            tmp.multiply(tmp1);
            ans.addPoly(tmp,1);
        }
    //然后对cos求导，符号会发生改变，sin少了1，cos多了1
        for (int i = 0;i < cosContent.size();i++) {
            Mono monoI = new Mono();
            monoI.addSinContent(this.cosContent.get(i));
            for (int j = 0;j < this.sinContent.size();j++) {
                monoI.addSinContent(this.sinContent.get(j));
            }
            for (int j = 0;j < this.cosContent.size();j++) {
                if (j != i) {
                    monoI.addCosContent(this.cosContent.get(j));
                }
            }
            monoI.set(this.xx,this.yy,this.zz,0 - this.sign,this.flg,this.num);
            Poly tmp = new Poly();
            tmp.addMono(monoI);
            Poly tmp1 = new Poly();
            tmp1.addPoly(this.cosContent.get(i),1);
            tmp1.devX();
            tmp.multiply(tmp1);
            ans.addPoly(tmp,1);
        }
        return ans;
    }

五、优缺点分析

优点

1.高内聚低耦合在本代码中的体现：

高内聚：每个类都有自己很全的方法，可以从上图中看出，poly类有自己的ToString函数，dev函数，mono类也是，也就是说，每个类的功能都很全

低耦合：类和类之间的功能相互干涉较少，比如re一定是递归下降的错误，输出格式错误就去检查mono和poly的ToString函数，求导错误就去检查求导函数，只要修改好了一个函数的错误就够了，不需要再顺带修改其他，模块和模块之间相互独立

2.ToString函数给我带来的简便性

方便对拍，因为不同形式的输入都可以根据字典序排序变成相同的化简顺序

容易合并同类项，我们可以实现string和poly的互换，也就可以做更深层次的合并

缺点

1.某个类的某个方法会被调用太多次

在复杂度分析表中可以看出，比较接口和ToString函数被调用的次数过多，因此只要有一个微小的错误，就极有可能影响到很多地方。为此，我想到的解决思路是统一在某个阶段进行合并化简，而不是进行边合并边化简的架构，但同时也可能带来Java heap等问题，还需要寻找一个折中的思路

2.递归下降的条件过于复杂

一个优美的递归下降不应该出现那么多的if语句
