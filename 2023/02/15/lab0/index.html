git学习记录

git怎么上传本地文件到仓库

cd D:\桌面 //输入你想上传的文件的目录
git init 
git add .("."代表该目录下的所有文件)
如果不想上传所有文件，把"."换成文件名，例如
git add 社会实践.docx

git文件的几种状态

git常用命令

git init  把当前目录变成一个空的git仓库
git clone 克隆一个已有的存储库到当前目录下
git status  查看当前分支的状态
git add (.或者<filename>) 加入暂存区
git restore (--staged <filename>)回退到上一个commit的状态
git checkout lab<x>  切换到相应的分支，需要保证目前所有文件的状态都是“未修改”
git commit -m <message>
git push 将本地的commit推送到一个远程仓库
git pull 将远程新建的分支下载到本地

文件操作

touch
用法：touch [选项] 文件名
作用：当文件存在时更新文件的时间戳，当文件不存在时创建新文件。

rm
用法：rm [选项] 文件
作用：删除文件。
选项（常用）：
-r              递归删除目录及其内容，删除非空目录必须有此选项，否则无法删除。
-f              强制删除，不提示用户确认，忽略不存在的目录。
-i              逐一提示用户确认每个将要被删除的文件。

diff
用法：diff [选项] 文件1 文件2
选项（常用）：
-b              不检查空白字符的不同。
-B              不检查空行。
-q              仅显示有无差异，不显示详细信息。

sed

请注意，当不加 -i 选项时，只在控制台显示编辑后的文件内容，而不修改文件本身。下面举几个使用例子，以便更好地理解 sed 命令的使用方法。

sed -n '3p' my.txt > test
#输出my.txt的第三行到test中

sed '2,5d' my.txt
#删除my.txt中的第二行到第五行
sed '2,$d' my.txt
#删除从第二行到最后一行

sed '2a drink tea'
#在第2行后添加这一行
sed '2i drink tea'
#在第2行前添加这一行
sed '2a drink tea or \drink bear?'
#在第2行后添加两行字

sed -e '4astr ' -e 's/str/aaa/' my.txt
#-e 选项允许在同一行里执行多条命令。例子的第一条是第四行后添加一个 str，
# 第二个命令是将 str 替换为 aaa。命令的执行顺序对结果有影响

sed 's/str1/str2/g' my.txt
# 在整行范围内把 str1 替换为 str2。
# 如果没有 g 标记，则只有每行第一个匹配的 str1 被替换成 str2
sed -i 's/oo/kk/g' ./test*
# 批量操作当前目录下以 test 开头的文件：

sed '2,5c No 2-5 number'
# 将第 2-5 行的内容取代成为 No 2-5 number 

sed -n '/oo/p'
#搜索有 oo 关键字的行
sed  '/oo/d'
#删除所有包含 oo 的行，其他行输出

grep

一个比较方便高效的查找操作罢了

grep -n -i "$2" ./"$1" > ref
-a              不忽略二进制数据进行搜索。
-i              忽略大小写差异。
-r              从目录中递归查找。
-n              显示行号。

awk

类似于printf，可以对文件中的内容进行标准化输入输出

awk -F ":" 'print $1' ./ref > ./"$3"
